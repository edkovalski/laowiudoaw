# Free Fire Diamond Top-up Bot

Telegram-бот для сервиса пополнения алмазов во Free Fire.

## 🚀 Установка и запуск

1. **Клонирование репозитория:**
   ```bash
   git clone <repository-url>
   cd bot
   ```

2. **Установка зависимостей:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Настройка конфигурации:**
   ```bash
   cp .env.example .env
   ```
   
   Отредактируйте файл `.env`:
   ```env
   BOT_TOKEN=your_bot_token_here
   ADMIN_ID=your_admin_id_here
   DATABASE_PATH=bot.db
   PAYMENT_DETAILS=Переведите [сумма] на карту/кошелек [реквизиты]
   ```

4. **Запуск бота:**
   ```bash
   python main.py
   ```

## 📋 Функционал

### Для пользователей:
- Команда `/start` - приветствие и начало заказа
- Выбор пакета алмазов (100, 310, 520, 1060, 2180)
- Ввод ID игрока с валидацией (8-12 цифр)
- Получение реквизитов для оплаты
- Отправка скриншота оплаты
- Получение уведомлений о статусе заказа

### Для администратора:
- Уведомления о новых заказах
- Просмотр скриншотов оплаты
- Кнопки "Выполнено" и "Отклонить" для управления заказами
- Возможность указать причину отклонения
- Команда `/admin` для просмотра ожидающих заказов

## 🏗️ Структура проекта

```
bot/
├── main.py                 # Основной файл бота
├── config.py              # Конфигурация
├── database.py            # Работа с базой данных
├── requirements.txt       # Зависимости
├── .env.example          # Пример конфигурации
├── handlers/
│   ├── user_handlers.py   # Обработчики пользователей
│   └── admin_handlers.py  # Обработчики администратора
├── keyboards/
│   └── inline_keyboards.py # Inline-клавиатуры
├── fsm/
│   └── states.py          # FSM состояния
└── services/
    └── order_service.py   # Бизнес-логика заказов
```

## 🗄️ База данных

Используется SQLite с двумя таблицами:
- `users` - информация о пользователях
- `orders` - информация о заказах

## 🔧 Технологии

- **Python 3.10+**
- **aiogram 3.x** - фреймворк для Telegram ботов
- **SQLAlchemy** - ORM для работы с базой данных
- **aiosqlite** - асинхронный драйвер SQLite
- **python-dotenv** - управление переменными окружения

## 📝 Примечания

- Бот использует FSM (Finite State Machine) для управления состоянием заказов
- Все операции с базой данных выполняются асинхронно
- Реализована валидация ID игрока (только цифры, 8-12 символов)
- Поддерживается отправка фото в качестве доказательства оплаты
